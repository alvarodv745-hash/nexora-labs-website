## 2. CSS Rules

- NEVER modify style.css unless explicitly instructed.
- NEVER rename existing CSS classes.
- NEVER delete existing CSS classes.
- Do NOT create new classes unless strictly necessary.
- Reuse existing design system whenever possible.

## 3. HTML Rules

- Maintain semantic structure (header, nav, section, footer).
- Do not remove content unless explicitly instructed.
- Do not summarize or simplify copy.
- Do not inject placeholder content.
- Do not add lorem ipsum.
- Keep accessibility attributes intact.

## 4. Structural Rules

- Prefer refactoring over rewriting.
- Prefer reordering over recreating.
- Do not duplicate sections.
- Maintain clean indentation.
- Output complete files when requested.

## 5. Navigation Rules

- Use anchor navigation for one-page structure.
- IDs must match navigation links exactly.
- Do not break existing scroll or JS behavior.

## 6. Output Rules
